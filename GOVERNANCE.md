# Governance

QMS is an Apache-2.0 open-source project. The project is maintained in public, and contributors retain copyright to their contributions while licensing them under the repository license.

## Decision Making

Maintainers make final decisions for releases, roadmap priority, and repository administration. Decisions should be explained in public issues, pull requests, or discussions whenever practical.

## Contribution Model

QMS uses the Developer Certificate of Origin signoff model instead of a contributor license agreement. Contributors certify their right to submit work by adding a `Signed-off-by` line to each commit.

## Project Principles

- Keep the self-hosted core useful without proprietary services.
- Prefer simple deployment paths that small organizations can run.
- Treat Arabic/English, RTL/LTR, accessibility, and data minimization as core product concerns.
- Keep commercial options, if any, additive rather than required for the open-source core.
